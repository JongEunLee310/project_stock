## Suggestions

### 1. 상태 전이 순서 미강제 (Minor / 설계 합치)

**위치:** app/domains/alert_candidates/service.py:51-74

`mark_read`/`confirm`은 현재 상태와 무관하게 목표 상태로 덮어쓴다(예: UNREAD에서 바로 CONFIRMED 가능, CONFIRMED→READ 역행 가능). 설계문서(035)의 표기는 `UNREAD → READ → CONFIRMED`지만, 요구사항은 "read API로 READ, confirm API로 CONFIRMED"로 각 API의 결과 상태만 규정하므로 현재 구현은 요구사항을 충족한다. 엄격한 단방향 전이가 필요한 도메인 규칙이 아니라면 현 구현으로 충분하다.

**권장 수정 방향:** 현 단계 조치 불필요. 향후 "확인된 후보의 재읽음 차단" 같은 규칙이 필요해지면 전이 가드 추가 검토. 단방향 전이를 의도했다면 설계문서 Decisions에 "전이 순서 비강제" 한 줄 명시 권장.
