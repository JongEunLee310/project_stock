## Suggestions

1. **(Minor) 사용되지 않는 메서드** — `NewsItemRepository.get_by_url`(repository.py:21)가 추가됐으나 어디서도 호출되지 않음. 서비스가 사용하는 것은 `exists_by_url`(NewsItem)과 `RawNewsEventRepository.get_by_url`뿐. 태스크는 "`exists_by_url` 또는 `get_by_url`" 중 하나만 요구했으므로 미사용 메서드는 제거 권장(또는 향후 사용 예정이면 유지 근거 명시).

2. **(Minor) 부분 실패 metadata 기록 경로 미테스트** — `analyze_watchlist_job`의 `_record_partial_failures`를 통해 JobRun.metadata에 부분 실패가 기록되는 경로는 단위 테스트로 커버되지 않음(서비스 레벨 `failures` 집계와 Job 성공/실패만 검증). `metadata_` 재할당이 SQLAlchemy dirty 감지 후 `succeed`의 commit에서 영속되는 구조는 정상이나, Job 경유 부분 실패 케이스 1개 추가 시 회귀 방지에 도움.

3. **(참고) 종목 내 다단계 commit 후 중간 실패** — `_process_asset`은 수집/요약/리포트/시그널/알림에서 여러 번 commit 발생. fetch 이후 단계에서 예외가 나면 `db.rollback()`은 마지막 commit 이후만 되돌리므로 해당 asset의 일부 데이터가 잔존할 수 있음. 현재 테스트는 fetch 단계 실패만 검증. 부분 실패 격리의 멱등성 강화는 후속 이슈에서 고려.
