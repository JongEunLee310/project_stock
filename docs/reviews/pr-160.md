# PR #160 — AssetBriefResponse에 market 노출 (feat/asset-brief-market)

리뷰어: Claude Code (Opus, 오케스트레이터). 구현: Codex. 브랜치: feat/asset-brief-market → main. 관련: BE #159, 설계 `asset-brief-market`, unblock FE #90·#95.

## Review Summary

설계 `docs/designs/asset-brief-market.md`대로 공유 계약 객체 `AssetBriefResponse`
(`app/domains/watchlists/schema.py`)에 `market: str` 필수 필드를 추가하고, 이 객체를
구성하는 3개 도메인 서비스(signals·watchlists·alert_candidates)의 생성부에서
`market=asset.market`를 전달합니다. asset 모델이 이미 market 컬럼을 보유하므로 스키마
마이그레이션은 없으며, 응답에 필드를 더하는 후방호환 additive 확장입니다.

변경은 총 9줄(스키마 1·서비스 3·테스트 5)로 최소이며, 세 생성부 모두 asset 객체가 이미
스코프에 있어 추가 조회·조인 없이 `asset.market`을 전달합니다. 기존 brief 필드
(symbol·name·price·change_percent·sector) 동작은 변경되지 않았습니다.

설계의 결정이 모두 반영됐습니다: (1) 공유 단일 정의 확장(도메인별 중복 정의 없음),
(2) 필수 필드로 추가(세 생성부 asset 접근 보장), (3) 세 도메인 응답 동시 반영,
(4) 계약 테스트·세 도메인 expand 테스트 갱신.

None.

추가하면 좋으나, 본 PR의 코드·테스트 정확성과는 무관하여 비차단입니다.
