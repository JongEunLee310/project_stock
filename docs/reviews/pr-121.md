# 로컬 리뷰: PR #121 — bcrypt 4.0.1 핀으로 passlib 호환성 복구

리뷰어: Claude Code (Opus, BE 트랙). 구현: Claude Code(사람 지시). 판정: **머지 가능(사람 최종 승인·머지 대기)**.

## 배경

로컬 Docker(PostgreSQL) smoke test 중 `POST /auth/register`·`/login`이 500 실패.
원인은 `passlib 1.7.4` + `bcrypt 5.0.0` 비호환 — passlib bcrypt 백엔드 초기화
자기검증(`detect_wrap_bug`)에서 `ValueError: password cannot be longer than 72 bytes`.
`pyproject.toml`에 bcrypt 핀이 없어 5.0.0이 설치된 것이 발단.

## 범위 대조

스코프: 의존성 핀 + 로컬 구동 보조. 애플리케이션 코드(보안 모듈·서비스·라우터) 무변경.
변경 4파일만, 범위 밖(passlib 제거/대체, security.py 리팩터링, 마이그레이션) 미포함 확인.

## 정확성

- **핀 범위**: `bcrypt>=4.0.1,<4.1`. 4.1+는 passlib 1.7.4와 `__about__` 제거 경고·동작 변화가 있어 4.0.x가 호환 마지막 라인. 적절. ✔
- **lock 일관성**: `uv.lock` bcrypt 5.0.0 → 4.0.1, 변경은 bcrypt 항목·specifier만(diff 확인). 타 패키지 영향 없음. ✔
- **익명 볼륨 `/app/.venv`**: 바인드 마운트(`.:/app`)가 호스트 macOS `.venv`를 컨테이너 linux `.venv` 위에 덮어쓰는 문제를 마스킹. 표준 패턴. ✔
- **.dockerignore**: 빌드 `COPY . .` 시 호스트 `.venv`/캐시 오염 차단. `.env`도 제외(런타임은 env_file/environment로 주입). ✔

## 비차단 관찰(후속)

- **근본 해소**: passlib 1.7.4는 2020년 마지막 릴리스로 유지보수 종료 상태. 중기적으로 `bcrypt` 직접 사용 또는 `pwdlib` 이관 검토 → bcrypt 상한 핀 제거 가능.
- **bcrypt 72바이트 한계**: 5.x는 72바이트 초과 비번에 ValueError. 4.0.x는 truncate. 현재 검증 로직에 길이 가드 없음 — 긴 비번 입력 시 동작 차이 인지 필요(후속 검토).
