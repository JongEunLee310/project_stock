## Review Summary

`GET /api/v1/portfolios/{id}/summary` 응답에 `risk_exposures`를 추가하는 구현으로, 설계 049 계약과 일치합니다. 신규 스키마 `RiskExposure`(`code`/`label`/`level`/`description`)를 `PortfolioSummaryResponse` 끝에 추가했고, 기존 필드·순서는 그대로입니다. 판정은 모두 `_build_summary`가 이미 보유한 값(`sector_weights`·`position_weights`·`cash_weight`·`total_value`·임계값)을 `_calculate_risk_exposures`에 넘겨 산출하므로 추가 시세 호출이 없습니다(`get_quote` 1회 유지). 섹터 쏠림은 `UNKNOWN`을 제외하고, 단일 종목 쏠림은 `assets_by_id`에서 symbol을 얻으며, 현금 부족은 `total_value == 0`(빈 포트폴리오)일 때 발생시키지 않아 false positive를 막습니다. `level`은 임계값의 1.5배 경계로 `HIGH`/`MEDIUM`을 가르고, 출력은 설계 §3.3의 결정적 순서(섹터 weight 내림차순 → 단일 종목 weight 내림차순 → 현금)로 정렬합니다.
