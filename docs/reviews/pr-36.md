## Suggestions

### 1. `_parse_json_list` 비리스트 JSON 무시 (Minor)

**위치:** `app/domains/theses/conflict_service.py:44-50`

JSON 파싱 후 `isinstance(parsed, list)` 검사로 객체·원시값이 들어오면 빈 리스트를 반환한다. 현재 유일한 쓰기 경로인 `NewsItemRepository.update_summary()`가 항상 리스트를 직렬화하므로 실제 도달할 수 없으나, 향후 다른 경로에서 쓸 경우 조용히 데이터를 버릴 수 있다.

**권장 수정 방향:** 유지해도 무방하나(방어적 패턴), 의도 명시 주석을 추가하는 방향도 고려.

### 2. `test_create_research_report_success` id 하드코딩 (Minor)

**위치:** `tests/test_research_reports.py:84`

`assert data["id"] == 1` — SQLite in-memory fixture가 테스트마다 완전히 재생성되므로 현재 문제없으나, 테스트 격리 방식이 변경될 경우 취약해진다.

**권장 수정 방향:** `assert isinstance(data["id"], int)` 또는 `assert data["id"] > 0`으로 교체 고려.
