- `app/adapters/llm/types.py`: enum 5종(`LLMTaskType`/`SensitivityLevel`/`RiskLevel`/`CachePolicy`/`ValidationStatus`)과 frozen dataclass 3종(`TokenUsage`/`LLMRequest`/`LLMResponse`). enum 값은 repo 컨벤션(`(str, Enum)`+UPPER_SNAKE)을 따르고, 가변 기본값은 `field(default_factory=dict)`로 안전하게 처리했습니다.

- `app/adapters/llm/__init__.py`: 신규 타입·provider를 기존 `__all__` 스타일로 재노출.
- 테스트 2종: enum 멤버·값 계약, frozen 불변성, 기본값, 가변 기본값 비공유, stub의 `NotImplementedError`.

## Suggestions

**S1 · `LLMRequest` 필수 필드 선택** — `system_prompt`/`sensitivity_level`/`risk_level`은 설계에 기본값이 명시되지 않아 Codex가 필수 필드로 두었습니다. 합리적 판단입니다. 다만 frozen dataclass에서 기본값 있는 필드(`input_payload` 등) 뒤에 필수 필드를 둘 수 없어, 필수 필드가 앞쪽에 모이는 필드 순서가 됐습니다. 동작상 문제는 없고, 호출부가 생기는 #134~#136에서 키워드 인자 사용을 권장합니다. 비차단.

**S2 · Phase 2 placeholder enum** — `CachePolicy`/`ValidationStatus`는 Phase 1에서 기본값(`BYPASS`/`NOT_VALIDATED`)으로만 흐르며 분기 소비처가 없습니다. 캐시·검증 구현(#137) 전까지 봉투 호환성을 위한 자리이며 설계 §3.4·§5에 명시돼 있습니다. 비차단.
