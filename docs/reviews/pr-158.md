# PR #158 — watchlist AI 관찰 메모 endpoint (feat/be-watchlist-observations)

리뷰어: Claude Code (Opus, 오케스트레이터). 구현: Codex. 브랜치: feat/be-watchlist-observations → main. 관련: BE #153, FE #86, 설계 060, ADR-009.

## Review Summary

설계 060대로 `GET /watchlists/{watchlist_id}/observations` endpoint를 추가했습니다. 인증 사용자 소유의 관심 목록에 대해 종목 현황·유의 사항을 LLM으로 on-demand 생성하고 공통 envelope(`ApiResponse[WatchlistObservationsResponse]`)로 반환하며, 결과를 영속화하지 않습니다.

`WatchlistObservationsService.generate`는 대시보드 브리핑(`DashboardBriefingService`) 패턴을 watchlist 스코프로 좁혀 재현합니다. `_get_owned_watchlist`로 소유권을 검증(미존재 404 `WATCHLIST_NOT_FOUND`, 비소유 403 `WATCHLIST_FORBIDDEN`)한 뒤, `list_by_watchlist(sort="priority")` → asset_id dedup → 상위 30개(`OBSERVATION_ITEM_LIMIT`) cap → `active_signal_types_by_asset`·`resolve_watchlist_status`·market quote로 `WatchlistHighlight` 리스트를 해소합니다. 이를 `to_watchlist_observation_snapshot`으로 CloudSafe projection(`WatchlistObservationSnapshot`, `AGGREGATED`)으로 만들어 `gateway.complete_json(LLMTaskType.WATCHLIST_NOTE, ...)`에 전달하고, `ObservationsResult.model_validate(...)` → `WatchlistObservationsResponse(**result.model_dump(), generated_at=utc_now())`로 매핑합니다.

설계 §1.1의 확정 결정이 모두 반영됐습니다: (A) 항목 타입은 059의 `WatchlistHighlight` 재사용, (B) 항목 수 상한 30, (C) LLM 출력 스키마(`ObservationsResult`, adapter)와 API 응답 스키마(`WatchlistObservationsResponse`, domain) 분리, (D) 소유권 에러 404/403 구분.

## Suggestions

**S1 · 종목 해소 로직 중복 (비차단, 설계에서 이미 인지)**

`_build_watchlist_highlights`·`_dedupe_top_watchlist_items`는 `DashboardBriefingService`의 동명 메서드와 거의 동일합니다(차이는 user 전체 vs watchlist 스코프·cap 값). 설계 060 §2가 공통 helper 추출을 의도적으로 후속으로 분리했고 핸드오프도 "억지로 공유 함수로 리팩터하지 말 것"으로 지시했으므로 스코프 내입니다. 다만 두 소비처가 확정된 지금, `WatchlistHighlight` 해소를 공용 resolver로 추출하면 중복이 해소됩니다. 세 번째 소비처가 생기거나 해소 규칙이 바뀌는 시점의 후속 개선 여지입니다. 현 단계 비차단입니다.

## Privacy Boundary (ADR-009)

- `WatchlistObservationSnapshot`은 `watchlist_id`·`item_count`·`items(WatchlistHighlight)`만 담고 금액 필드(보유 수량·평가액)를 포함하지 않습니다.
- `sensitivity = AGGREGATED`이며 `PrivacyGate.guard`를 통과합니다.
- 항목 수 상한 30으로 단일 cloud 호출 payload를 bounding합니다.
- 회귀 방지 테스트: projection에 금액 필드 부재 단언, 원본 entity의 `PrivacyGate` 거부 단언.
