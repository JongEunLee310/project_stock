## Suggestions

### 1. 구조만 고정하고 값은 고정하지 않음 (Minor)

**위치:** `tests/test_api_contract.py`

`assert_contract`가 필수 키 존재 + 타입만 검사하고 응답 값은 단언하지 않는다(`meta` 카운트 단언은 시드 1건 기준의 결정적 값이라 예외적으로 안전). 설계 041 Decisions의 "구조만 고정, 값 미고정" 원칙을 정확히 따른다 — 시드/시간 의존 flaky를 피하는 적절한 선택.

**권장 수정 방향:** 유지.

### 2. nullable 필드를 `(T, type(None))` 튜플로 표현 (Minor)

**위치:** `tests/test_api_contract.py:89-172`

`reason`/`memo`/`sector` 등 nullable 필드를 `(str, type(None))` 형태로 두어 `isinstance` 검사에 None 허용을 명시한다. 외부 라이브러리 없이 nullable contract를 표현하는 간결한 방식 — 설계의 "의존성 최소화" 의도와 일치.

**권장 수정 방향:** 유지.
