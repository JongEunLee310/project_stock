## Review Summary

`GET /api/v1/portfolios/{id}/summary` 응답에 `day_change_value` / `day_change_percent`를 추가하는 구현으로, 설계 048의 계약과 완전히 일치한다. `_get_quotes_by_symbol` 반환형을 `dict[str, QuoteResult]`로 확장하여 `change_percent`를 재사용하고, `_calculate_position_day_change` 헬퍼에서 `prev_value = market_value / (1 + change_percent/100)` 수식으로 포지션별 일간 변동액을 산출한다. 포지션 합계 `day_change_value`와 `prev_total_value = total_value - day_change_value`를 분모로 한 `day_change_percent` 계산도 설계 §3 정본과 동일하다. 외부 호출 횟수는 변경 없이 `get_quote` 1회이며, `RecordingMarketProvider`가 이를 단언으로 검증한다. price == 0 및 분모 0 가드 모두 코드에 구현되어 있다.

**S1 · 분모 0 전용 테스트 부재**
