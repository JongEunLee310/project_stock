## Review Summary

`GET /api/v1/portfolios/{id}/summary` 응답에 `day_change_value`·`day_change_percent`를 추가하는 구현으로, 설계 048 계약과 일치합니다. `_get_quotes_by_symbol` 반환형을 `dict[str, QuoteResult]`로 확장해 `change_percent`를 재사용하고, `_calculate_position_day_change` 헬퍼에서 `prev_value = market_value / (1 + change_percent/100)` 수식으로 포지션별 일간 변동액을 산출합니다. 포지션 합계 `day_change_value`와 `prev_total_value = total_value - day_change_value`를 분모로 한 `day_change_percent` 계산도 설계 §3 정본과 동일합니다. 외부 호출은 기존과 같이 `get_quote` 1회이며, `RecordingMarketProvider`가 이를 단언으로 검증합니다. price == 0과 분모 0 가드도 모두 구현되어 있습니다.

## Suggestions

**S1 · 분모 0 전용 테스트 부재**

`change_percent = -100`(전일 대비 100% 하락, 분모 = 0) 케이스를 직접 단언하는 전용 테스트는 아직 없습니다. 코드의 `denominator == 0` 가드가 정상 동작해 동작상 문제는 없지만, 이 경계 케이스를 명시적으로 확인하는 테스트를 후속에 추가하면 좋겠습니다. 비차단입니다.
