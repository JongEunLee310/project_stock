## Suggestions

### 1. 신규 에러 테스트가 기존 패턴과 일관됨 (Minor)

**위치:** `tests/test_error_responses.py`

`api_error` 헬퍼로 envelope `{code, message}`(+422 시 `fields`)를 검증하는 방식이 기존 `test_error_handlers`/도메인 테스트 패턴과 동일하다. SQLite in-memory + dependency override 기반 `conftest` 픽스처를 재사용하며 신규 픽스처 구조를 도입하지 않았다 — 설계 040 Decisions 준수.

**권장 수정 방향:** 유지.

### 2. 보강 테스트가 응답 구조를 고정하지 않음 (Minor)

**위치:** `tests/test_alert_candidates.py`, `tests/test_portfolios.py`

미존재 경로 단언이 error `code`/`message`만 검사하고 전체 응답 스냅샷은 만들지 않는다. 스냅샷/contract 고정은 #61 범위로 명시되어 있어 이번 범위와 정확히 분리됨 — 적절.

**권장 수정 방향:** 유지.
