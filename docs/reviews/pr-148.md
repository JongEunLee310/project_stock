# PR #148 로컬 리뷰 — #135 PrivacyGate + CloudSafe projection 경계

## Review Summary

ADR-009가 확정한 클라우드 데이터 경계를 처음으로 코드로 옮긴 PR입니다. 핵심은 두 장치입니다. 하나는 클라우드로 나갈 수 있는 페이로드의 공통 base 타입 `CloudSafePayload`이고, 다른 하나는 경계에서 타입과 민감도를 검사해 위반을 거부하는 `PrivacyGate`입니다. 첫 구체 projection `PortfolioConcentrationSnapshot`과 원본 entity에서 그것을 파생하는 `to_concentration_snapshot()`도 함께 들어왔습니다.

설계 `docs/designs/055-privacy-gate-cloudsafe.md`와 구현이 정확히 일치합니다. whitelist(허용 필드만 담는 별도 타입) 방식을 지켰고 redaction/denylist는 쓰지 않았습니다. `PrivacyGate.guard`는 `CloudSafePayload` 인스턴스가 아니거나 민감도가 `AGGREGATED`/`PUBLIC`이 아니면 거부하는 fail-closed 동작입니다. projection은 정확 수치 대신 구간(band)과 불리언만 담아 절대 수량·잔액·`user_id`·종목 식별자를 노출하지 않습니다.

#133 타입·#134 라우터와 같은 "도입 후 소비는 다음 이슈" 패턴입니다. PrivacyGate를 실제 클라우드 분기에 거는 배선은 #136(gateway)이 소유하며 본 PR 범위가 아닙니다.

## Blocking

None.

## Suggestions

- `to_concentration_snapshot`의 구간 경계값(`POSITION_COUNT_*`, `*_SHARE_MAX`)은 모듈 상수로 잘 분리되어 있습니다. 다만 ADR-009가 지적한 소규모·엣지 포트폴리오 재식별 방지는 아직 반영되지 않았습니다. 설계에도 Phase 2(브리핑 기능 설계)로 이연한다고 명시되어 있으니 본 PR에서 다룰 사항은 아니며, #136 이후 브리핑 기능을 붙일 때 집계 정밀화를 함께 검토하면 됩니다.
- `_position_count_band`는 보유 종목이 0개일 때도 `"1-5"`를 돌려줍니다. 현재 호출 경로가 없어 실제 영향은 없지만, gateway가 이 projection을 소비하는 #136 단계에서 빈 포트폴리오를 어떻게 표현할지 한 번 짚어두면 좋겠습니다.
