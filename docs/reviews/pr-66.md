## Suggestions

### 1. FastAPI import 분리 (Minor)

**위치:** `app/api/v1/endpoints/assets.py:3-4` (및 동일 패턴이 있는 타 엔드포인트)

`from fastapi import APIRouter, Depends`와 `from fastapi import Query`가 두 줄로 분리되어 있다. 동작·린트에는 문제없으나 한 줄(`from fastapi import APIRouter, Depends, Query`)로 병합하면 기존 스타일과 일관된다.

**권장 수정 방향:** import 한 줄 병합. (#47 작업 시 함께 정리해도 무방.)

### 2. 목록 count 추가 쿼리 (Minor)

**위치:** 목록 엔드포인트 전반 (예: `app/domains/assets/repository.py` `count_all`)

페이지네이션 `total` 산출을 위해 목록 요청마다 `count()` 쿼리가 1회 추가된다. 현재 데이터 규모에서는 무해하며 정확한 total을 제공하는 올바른 구현이다. 향후 대용량 테이블에서 성능 이슈가 보이면 keyset 페이지네이션 등을 검토.

**권장 수정 방향:** 현 단계 수정 불필요. 향후 성능 관찰 포인트로만 기록.
