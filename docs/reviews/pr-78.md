## Suggestions

### 1. `APP_ENV`가 정의만 되고 아직 소비되지 않음 (Minor)

**위치:** `app/core/config.py:9`

`APP_ENV`는 설정에 추가됐으나 현재 코드 어디에서도 분기에 사용되지 않는다. 이슈 완료 조건("개발/테스트/운영 구분 가능")상 정의 자체로 충분하며 설계 038에서 환경별 서브클래스를 의도적으로 배제했으므로 문제 없다. 다만 향후 환경별 동작(예: prod에서 placeholder SECRET_KEY 경고, 디버그 토글)을 붙일 자리임을 인지. 지금 조치는 불필요.

**권장 수정 방향:** 현 상태 유지. 환경별 분기가 실제로 필요해지는 시점(#57 CORS 또는 운영 배포)에 소비 지점 추가.

### 2. `.env.example` 키 일치 테스트의 결합도 (Minor)

**위치:** `tests/test_config.py:71`

`test_env_example_keys_match_settings_fields`가 `.env.example`와 `Settings.model_fields`의 완전 일치를 강제한다. 매우 좋은 가드지만, #57에서 `CORS_ORIGINS` 등을 추가할 때 양쪽을 동시에 갱신하지 않으면 실패한다. 의도된 동작이므로 수정 불필요 — #57 핸드오프에 "`.env.example`도 함께 갱신" 요구가 이미 포함됨을 확인.

**권장 수정 방향:** 유지. 후속 작업자에게 이 테스트의 존재를 상기(설계 037/핸드오프에 반영됨).
