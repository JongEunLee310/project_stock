## Suggestions

1. **(Minor) `stable_password_hashing` autouse 픽스처 중복** — `test_auth.py`와 `test_integration_flow.py`에 동일 픽스처가 복제됨. conftest 중앙화 취지와 상충하므로 `conftest.py`로 이전 가능. 기능 영향 없음(비차단).
2. **(참고) 실제 패스워드 해싱 경로는 미검증** — 위 픽스처가 `hash_password`/`verify_password`를 스텁 처리해 passlib 실해싱은 테스트에서 실행되지 않음. 인증 흐름(JWT 발급·검증)은 실제 경로로 검증되나, 해시 알고리즘 자체의 회귀는 잡지 못함. 속도·결정성 트레이드오프로 수용 가능하나, 향후 해싱 정책 변경 시 별도 단위 테스트 권장.

## 좋은 점

- 기존에 항상 override되던 `get_current_user`를 우회 없이 검증 — register/login/me의 실제 JWT 경로(무헤더·변조 토큰 401 포함)가 처음으로 커버됨.
- conftest 중앙화로 중복 -503 라인, 기존 테스트 함수 제거·약화 없이 순수 dedup(diff상 `def test_` 삭제 없음).
- 통합 테스트의 비중 산식 정확(AAPL 300 / 총 400 = 0.75 > 임계 0.6 → Signal 1건), 응답·조회 API 양쪽에서 동일 Signal 확인.
- 비동기 워커(뉴스→Alert) 흐름은 Redis 의존이라 범위에서 제외하고 서비스 계층 `test_analysis_flow.py`에 위임 — 경계 설정 합리적.
