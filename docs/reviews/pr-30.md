## Suggestions

### 1. `ThesisUpdate.summary`가 `None`을 허용해 NOT NULL 컬럼을 덮어쓸 수 있음 (Minor Bug)

**위치:** `app/domains/theses/schema.py:14` / `app/domains/theses/repository.py:60-70`

`ThesisUpdate.summary`는 `str | None = None`이고 `ThesisService.update()`에서 `data.model_dump(exclude_unset=True)`를 사용한다.

`exclude_unset=True`는 JSON 바디에서 명시적으로 전송된 필드만 포함한다. 따라서 클라이언트가 `{"summary": null}` 을 전송하면 `summary: None`이 업데이트 딕셔너리에 포함되어 `thesis.summary = None`이 실행되고, DB의 `NOT NULL` 제약을 위반해 500 오류가 발생한다.

**권장 수정:**
```python
# schema.py
class ThesisUpdate(BaseModel):
    summary: str | None = None  # 현재

# 방법 A: 타입을 str로 변경 (null을 전송할 이유가 없으므로)
    summary: str | None = None  → summary: str | None = None  # 유지하되 서비스에서 필터링

# 방법 B: 서비스에서 None 값 제거
    updated = self.repo.update(
        thesis.id,
        {k: v for k, v in data.model_dump(exclude_unset=True).items() if v is not None},
    )
```

방법 B가 더 안전하다. 단, `risk_factors`와 `invalidation_conditions`는 nullable이므로 의도적으로 None 설정이 가능해야 한다. 따라서 `summary` 필드만 타입을 `str`로 변경하는 방법이 가장 명확하다:

```python
class ThesisUpdate(BaseModel):
    summary: str | None = None        # str로 변경 권장
    risk_factors: str | None = None   # None 허용 유지
    invalidation_conditions: str | None = None  # None 허용 유지
```
