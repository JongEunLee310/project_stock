## Review Summary

`GET /api/v1/watchlists/{id}/items`에 선택적 `expand` query 추가. `expand=asset`일 때만 각 item에 `asset`(symbol/name/price/change_percent/sector) 포함, 그 외(`expand` 없음·미지원 값)는 기존 응답 그대로 유지해 **하위호환** 보존. 시세는 기존 `get_market_provider()`(asset detail이 쓰는 동일 경로) 재사용 — 새 외부연동·테이블 없음. 금액·비율은 문자열 Decimal(C5)로 일관. 스코프 4파일(endpoint/schema/service+전용 테스트), 보호 파일 변경 없음. 전용 테스트 파일로 `test_api_contract.py` 충돌 회피.

## Suggestions

1. **(성능 참고) N+1 시세 조회** — 항목별 `asset` 시세를 개별 조회하면 목록 길이에 비례한 provider 호출이 발생할 수 있음. 현재 mock provider는 비용이 낮아 무방하나, 실 provider 연동 시 symbol 목록 일괄 조회(batch)로 전환 권장(비차단).
