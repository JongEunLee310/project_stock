## Suggestions

### 1. `is_expired` 시점 일관성 (Minor)

**위치:** `app/domains/signals/schema.py:47-56`, `repository.py:61-63`

`SignalResponse.is_expired`와 `SignalRepository._active_clause`가 각각 별도로 `datetime.now(timezone.utc)`를 호출한다. 동작상 문제는 없으나, naive datetime을 UTC로 가정하는 응답 변환 로직과 DB 비교(`expires_at > now`)가 한 곳에 모여 있지 않아 향후 타임존 정책 변경 시 두 군데를 함께 고쳐야 한다. 현재 단계에서는 유지 무방.

### 2. `score` 범위 검증 부재 (Minor)

**위치:** `app/domains/signals/schema.py:15`

설계상 `score`는 0–100이나 `SignalCreate.score: int`에 범위 제약이 없다. Signal 생성은 현재 내부(Rule Engine) 호출이 주 경로이고 API는 인증된 사용자만 접근하므로 실질 위험은 낮다. 필요 시 `Field(ge=0, le=100)` 추가 고려.
