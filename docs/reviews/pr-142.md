## Review Summary

Epic #141 Phase 1의 선행 작업으로, 하이브리드 LLM 구조의 핵심 결정 3건을 구현 전에 ADR로 고정합니다(이슈 #132). 문서만 변경하며 코드·스키마·시세 호출은 없습니다.

- `ADR-007` Provider Abstraction: transport `LLMClient`(ABC)를 그대로 두고 상위 오케스트레이션 `LLMGateway`를 신설합니다. repo 컨벤션(`LLMClient`=transport, `XxxProvider`=factory 외부어댑터)에 맞춰 스펙의 `LLMProvider` 네이밍을 `LLMGateway`로 회피하고, 전 계층 sync 컨벤션에 따라 스펙의 `async def generate`를 기각합니다.
- `ADR-008` Task Routing Policy: `task_type → provider` 매핑을 config 주도로 두고(신규 `LLM_PROVIDER` 설정, 기존 `MARKET_PROVIDER` 패턴 답습), 초기 클라우드 우선·로컬 `future_primary`를 기록합니다. unknown task는 fail-closed(에러)로 처리해 미매핑 작업이 무성의하게 클라우드로 흐르는 것을 막습니다. 라우팅 표는 Epic #141을 정본으로 인용합니다.
- `ADR-009` Cloud Data Boundary / CloudSafe Projection: 원본 entity(Portfolio/Account/Holding/Trade) 클라우드 전송을 금지하고, redaction(denylist, fail-open)이 아니라 화이트리스트 projection(fail-closed)을 채택합니다. CloudSafe projection은 HTTP `schema`가 아니라 게이트웨이 경계 전용 타입입니다. 민감도 등급(RAW/SEMI/AGGREGATED/PUBLIC)을 라우팅·게이트 공통 어휘로 정의하고, 게이트웨이 단일 choke point에서 강제합니다.
- `docs/decisions/README.md` 인덱스에 누락돼 있던 ADR-006과 신규 ADR-007~009를 반영합니다.
