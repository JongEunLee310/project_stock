## 범위 확인

설계 064 §3~§4의 8개 항목 델타가 모두 반영되었습니다. 스코프를 벗어난 변경(도메인 규칙 변경,
DB 마이그레이션, envelope/pagination 변경, 보호 파일 변경)은 없습니다.

- `UserResponse`: `created_at` 노출 + `username` computed field(email local-part)
- `ResearchSummaryResponse`: 정성 형태로 교체, `ResearchSummarySource` 제거, `ResearchRisk` 신규
- `BuyChecklistItem`: `{id, label, description, checked}`로 정렬, 쓰기 경로 key 유지
- `AssetDetailResponse`: `market_cap`·`next_earnings_date` 추가, `as_of`→`updated_at`
- `ResearchReportResponse`·`ThesisResponse`: `title` computed field(summary 파생), report `source` 추가
- `AlertResponse`: `title` 파생(`symbol alert_type`)
- `AssetResponse`: `sector` 노출
- `QuoteResult`(+mock): `market_cap`·`next_earnings_date` 추가

## 잔여 참조 정리

- `ResearchSummarySource`·research-summary factor-list 필드: 코드·테스트·openapi 컴포넌트
  단언에서 모두 제거 확인.
- `as_of`: 외부 계약(`AssetDetailResponse`)에서 제거되고 `updated_at`으로 대체됨. 서비스의
  `updated_at=quote.as_of` 매핑과 provider 내부 `QuoteResult.as_of` 유지는 의도된 설계입니다.

## 관찰 사항 (비차단)

1. **`stance` 값 형태**: research-summary가 `BUY_CANDIDATE`·`WATCH` 같은 enum성 문자열을
   반환합니다. FE adapter는 `summary.stance`를 그대로 표시하므로, 원문 그대로 노출됩니다.
   한국어 라벨 매핑은 FE #101에서 처리할 항목입니다. BE 계약상 문제는 아닙니다.
2. **`title` 파생 fallback**: report/thesis title이 비어있을 때 영어 fallback
   (`Research report`·`Investment thesis`)을 사용합니다. mock 단계에서는 수용 가능하며, 실데이터
   컬럼 도입 시 재검토합니다.
3. **`decision_checklist/schema.py` 포맷**: `ChecklistItemKey` Literal과 `BuyChecklistItem`
   클래스 사이 빈 줄이 사라졌습니다. ruff 통과(E3xx 미적용)로 기능·CI 영향은 없습니다. 후속
   정리 시 여백을 복원하면 가독성이 개선됩니다.

## ADR / 실패 기록

- ADR 불필요: 기존 계약 스냅샷 테스트(설계 041)·envelope 규약을 따르는 필드 정렬입니다. 주요
  결정(정성 스키마 FE 형태 채택·파생값 정렬)은 설계 064에 기록되어 있습니다.
- 실패 기록 불필요.

FE #101(adapter fallback 제거·`stance` 라벨 매핑)로 이어집니다.
