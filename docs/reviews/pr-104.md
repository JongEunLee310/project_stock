## Review Summary

공통 `UtcDatetime` annotated 타입(`app/core/schema.py`)을 도입해 응답 스키마 datetime 필드를 UTC ISO 8601 `Z`로 일관 직렬화. `PlainSerializer(..., when_used="json")`이라 JSON 출력만 문자열 `Z`로 바뀌고 `model_dump(mode="python")`은 datetime 객체를 유지 — 내부 로직 영향 없음. naive datetime은 UTC로 간주해 SQLite 테스트 DB의 naive 반환 케이스도 일관 처리. 응답 모델 21개 필드(14개 파일)만 교체하고 입력 모델(`*Create`)은 유지. DB 스키마·Alembic 변경 없음. 스코프 내 파일만 수정, 보호 파일(.codex/docs/harness/docs/decisions/AGENTS/CLAUDE/workflows) 변경 없음.

## Suggestions

1. **(참고) per-field opt-in의 누락 위험** — `UtcDatetime`을 필드마다 명시 적용하는 방식이라, 향후 신규 응답 스키마가 plain `datetime`을 쓰면 규약에서 누락될 수 있다. 지금은 명시적이라 검토가 쉬워 적절하나, 응답 베이스 스키마나 lint 규칙으로 강제하는 방안은 후속 고려 가치 있음(비차단).
2. **(Minor) microsecond 표기** — `astimezone(UTC).isoformat()`은 microsecond가 있으면 `...123456Z`로 출력됨. 현재 소스 datetime에 microsecond가 드물어 문제 없으나, FE 파싱이 microsecond 유무 모두 허용하는지 어댑터(FE #45)에서 확인 권장(비차단).
