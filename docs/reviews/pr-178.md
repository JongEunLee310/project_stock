## Review Summary

LLM 데이터 파이프라인 1단계의 데이터 계약을 순수 정의로만 추가한 PR입니다. 신규 도메인 세 개가
계약을 나눠 담습니다. `app/domains/ingestion`은 두 수집 경로가 공유할 경계 projection
`RawProviderResponse`와 `ProcessingStatus`·`RawDataType` enum을 정의하고 저장 모델은 만들지
않습니다(Decision A). `app/domains/llm_context`는 `LLMContextBundle`과 중첩 projection 아홉 종,
`DataQualityStatus` enum을 정의하며, `data_quality`를 기본값 없는 필수 필드로 두어 누락 시
validation이 실패하도록 했습니다(지침 13절). `app/domains/llm_analysis`는 출력 스키마
`LLMAnalysisResult`와 영속 모델 `LLMAnalysisRun`을 담고, 결과 전용 테이블 대신 `output_json`
컬럼에 결과를 저장합니다(Decision B).

설계 의도가 정확히 반영됐습니다. `task_type`·`risk_level`은 `app/adapters/llm/types.py`의
`LLMTaskType`·`RiskLevel`을 그대로 재사용해 어댑터와 어휘를 통일했고(Decision C), `SuggestedAction`은
`buy_watch`/`hold`/`trim_watch`/`avoid`/`need_more_data`만 갖는 독립 enum으로 정의해 기존
`DecisionType`(BUY/SELL 포함)을 미러링하지 않습니다(Decision D, 지침 17절). `LLMAnalysisRun`의
`input_context_json`은 `nullable=False`로 두어 결과만 저장하는 구조를 계약 수준에서 차단합니다.
마이그레이션 `c3d4e5f60057`은 기존 head `f6005703a129` 뒤에 붙어 단일 head를 유지하고, 신규
모델은 `alembic/env.py`에 import 한 줄로 등록됐습니다. 보호 파일(`app/adapters/llm/*`, 기존 도메인
모델)은 건드리지 않았고, 허용된 예외인 `alembic/env.py` 한 줄만 수정됐습니다. 2~7단계 로직과
route는 포함되지 않아 범위를 정확히 지켰습니다.

## Suggestions

- **S1 · 마이그레이션 타임스탬프 default 표현**: `created_at`/`updated_at`이
  `server_default=sa.text("now()")`로 하드코딩됐습니다. 모델의 `TimestampMixin`(`func.now()`)과
  동작은 같지만, 다른 마이그레이션과의 표기 일관성은 후속에서 한 번 맞춰두면 좋겠습니다. 이번
  범위에서 수정할 필요는 없습니다.
- **S2 · 중첩 projection의 필수/옵셔널 정책**: `SymbolCard.portfolio_context`는 `| None`이되
  기본값이 없어 항상 명시 전달이 필요하고, 리스트 필드도 모두 required입니다. 계약을 엄격히
  잡은 선택으로 타당합니다. 다만 6단계 ContextBuilder에서 조립 편의를 위해 리스트에 기본
  빈 값을 줄지 여부는 그때 함께 판단하면 됩니다.
- **S3 · `RecentNewsItem.published_at` required**: 뉴스 원본은 `published_at`이 결측일 수 있는데
  projection은 이를 필수 datetime으로 둡니다. 3단계 Normalizer가 값을 채우거나 누락을 어떻게
  다룰지(예: 제외 또는 대체 시각)는 해당 단계에서 정하면 됩니다. 계약 자체는 문제없습니다.

## Questions

- **Q1 · `risk_level` 직렬화 표기**: 재사용한 `RiskLevel`은 대문자(`MEDIUM`)라, 지침 8절 예시의
  소문자(`medium`)와 표기가 다릅니다. 7단계 LLM Gateway에서 출력 계약을 확정할 때 대문자 그대로
  둘지, 소문자 alias가 필요할지 판단이 필요합니다. 1단계 계약으로는 재사용이 맞다고 봅니다.
