## Suggestions

1. **(참고) asset 단위 중복 방지의 다중 포트폴리오 한계** — `check_concentration`은 `exists_active(asset_id, RISK_ALERT, None)`로 중복을 막으나, Signal이 asset 단위(portfolio 식별 없음)라 같은 종목을 보유한 서로 다른 포트폴리오가 모두 초과 시 두 번째 포트폴리오의 점검에서는 이미 활성 신호가 있어 생성되지 않음. evidence에도 portfolio_id가 없어 어느 포트폴리오에서 발생했는지 식별 불가. MVP 범위에서는 수용 가능하나, 포트폴리오 단위 추적이 필요해지면 후속 이슈에서 보강 권장.
2. **(Minor) weight 정밀도 미고정** — `cost_value / total_cost_value` 결과를 quantize하지 않아 비정수 분할(예: 1/3) 시 응답에 28자리 소수가 노출될 수 있음. 테스트 케이스는 0.75/0.25로 깔끔하나, 응답 가독성을 위해 소수 자릿수 고정(quantize) 검토 가능. 기능 영향 없음(비차단).

## 좋은 점

- `get_summary`(읽기 전용)와 `check_concentration`(Signal 생성)을 명확히 분리해 GET의 부수효과를 제거. 설계 의도와 일치.
- 총 원가 0(보유 종목 없음) 시 비중 0 안전 처리, `zip(..., strict=True)`로 길이 불일치 방어.
- 마이그레이션이 server_default로 기존 행을 안전하게 채움(NOT NULL 추가 안전).
- 테스트 커버리지 충실: 비중 계산·빈 포지션·Signal 생성·중복 미생성·임계치 미달·소유권 403/미존재 404.
