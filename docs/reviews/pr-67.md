## Questions

### 1. `auto_error=False` 전환에 따른 `WWW-Authenticate` 헤더

`oauth2_scheme`을 `auto_error=False`로 바꿔 토큰 누락도 `AUTH_INVALID_TOKEN` envelope로 흐르게 한 처리는 적절하다. 다만 기존 FastAPI 기본 동작이 401에 부여하던 `WWW-Authenticate: Bearer` 헤더는 더 이상 부여되지 않는다. 순수 JSON API 클라이언트에는 영향이 없으나, 표준 OAuth2 흐름을 따르는 소비자가 있다면 확인이 필요하다. (현 프론트 연동 범위에서는 무해로 판단.)
