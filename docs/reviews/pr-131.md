## Review Summary

설계 052 계약대로 읽기 전용 집계 엔드포인트 `GET /api/v1/decision-logs/stats`를 추가했습니다. 응답 `DecisionLogStatsResponse`는 `decision_type_counts`(존재 유형만), `total`(counts 합), `recent_reviewed`(최근 검토 상위 5건)로 구성됩니다. `count_by_decision_type`는 `decision_type` GROUP BY count를, `list_recent_reviewed`는 `reviewed_at IS NOT NULL` 필터 + `reviewed_at` desc(+`id` desc tie-break) + limit를 수행합니다. `get_stats`는 두 리포지토리 결과를 조립하고 `total`을 counts 합으로 산출하며, 0건이면 빈 맵·빈 리스트·`total=0`을 반환합니다. `/stats` 라우트를 `/{decision_log_id}`보다 먼저 선언해 경로 충돌을 방지했고, `response_model`·기존 엔드포인트 동작은 변경하지 않았습니다. 모델/마이그레이션·시세 호출은 없습니다.

## Suggestions

**S1 · `recent_reviewed`의 표시 텍스트가 결정 시점 필드**

`ReviewedDecisionItem`은 `reason`/`risk_note`를 그대로 노출합니다. 이는 회고 반성문이 아니라 판단 시점 근거이며, 설계 052가 의도한 진실값 노출(회고 텍스트 미합성)과 일치합니다. 추후 별도 회고 메모 계약이 생기면 그때 분리하면 됩니다. 비차단 사항입니다.
