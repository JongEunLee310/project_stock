# PR #151 로컬 리뷰 — 포트폴리오·대시보드 AI 브리핑 + LLM Phase 2 ADR·설계

## Review Summary

LLMGateway의 첫 사용자 대면 소비처로 포트폴리오·대시보드 브리핑을 추가한 PR입니다. 두
브리핑은 입력 CloudSafe projection만 다르고, 공통 출력 스키마 `BriefingResult`와
`LLMGateway.complete_json` 경유 패턴을 공유합니다. 설계 057·058, deferred ADR-010/011/012,
핸드오프 task-057가 함께 들어 있습니다. 구현은 Codex가 핸드오프를 받아 main 기준 브랜치에서
수행했고, 본 리뷰는 그 결과를 로컬에서 점검한 기록입니다.

핵심 점검 대상은 ADR-009가 확정한 클라우드 데이터 경계입니다. 신규 projection
`PortfolioBriefingSnapshot`·`DashboardBriefingSnapshot`은 `sensitivity=AGGREGATED`이며, 비중·집계·
공개 지표·`symbol`만 담고 수량·평균단가·평가액·원가·절대 현금잔액은 담지 않습니다.
`to_briefing_snapshot`은 이미 집계된 `PortfolioSummaryResponse`에서 화이트리스트 필드만 옮기며
금액 필드를 끌어오지 않습니다. `symbol`·비중 포함은 사용자와 정렬한 trade-off로 057 §3.1에
근거가 기재되어 있습니다.

모든 LLM 호출은 게이트웨이를 거치고, 라우팅 표(`router.py`)는 수정하지 않았습니다(보호 범위
준수). 엔드포인트는 기존 인증·소유자 deps를 재사용하며 403/401/404를 테스트로 덮습니다.

## Suggestions

- 대시보드 `watchlist_highlights`는 빈 배열로 생략되었습니다. 기존 watchlist 도메인에 계약과
  맞는 `status` 필드가 없어 신규 도메인 금지 제약 안에서 채울 수 없었고, 핸드오프의 축소
  분기를 정확히 따른 결정입니다. 058 설계에 현황·후속을 반영했습니다. highlight를 채우려면
  watchlist 표시 상태 계약이 선행되어야 합니다.
- `PortfolioBriefingService.generate`가 `asset_id`마다 `AssetRepository.get_by_id`를 호출합니다.
  보유 종목 수가 작아 실제 영향은 미미하나, 종목 수가 커지면 batch 조회로 바꿀 여지가 있습니다.
- position 일간변화는 market provider의 `change_percent`를 사용합니다(mock이라 결정적). 가정이
  PR/커밋에 보고되어 있어 추적 가능합니다.

## Documentation Impact

설계 057·058과 핸드오프 task-057, deferred ADR-010/011/012가 PR에 포함되어 있습니다. 본 기능은
ADR-007/008/009의 기존 결정을 따르므로 신규 ADR은 불필요합니다. 본 구현이 살아있는 소비처가
되면서 ADR-010/011/012를 향후 `Accepted`로 확정하는 트리거가 되며, 그 확정은 별도 작업입니다.
Failure Record 사유는 없습니다.

## 머지 전 사람 리뷰 (ADR-009)

클라우드로 나가는 projection은 프라이버시 민감 경계입니다. ADR-009 Consequences가 이런 경계의
구현은 머지 전 명시적 사람 리뷰를 요구합니다. projection 화이트리스트가 금액·원본 entity를
정확히 배제하는지 개발자 확인을 거쳐 머지하시길 권합니다. 최종 머지 권한은 개발자에게 있습니다.
