## Suggestions

### 1. `create_app` 팩토리 도입에 따른 import 표면 변화 (Minor)

**위치:** `app/main.py`

기존 모듈 레벨 `app` 구성을 `create_app(app_settings=settings)` 팩토리로 리팩터했고 모듈 끝에서 `app = create_app()`을 노출한다. ASGI 진입점(`app.main:app`)은 그대로라 런타임 영향 없음. 테스트가 settings를 주입해 앱을 생성할 수 있게 된 좋은 변경. 설계 037에는 팩토리 전환이 명시되지 않았으나 "origin을 settings에서만 읽는다" 결정을 테스트 가능하게 구현한 합리적 선택 — 범위 내로 판단.

**권장 수정 방향:** 현 상태 유지.

### 2. `.env.example`의 CORS 기본값과 `Settings` 기본값 차이 (Minor)

**위치:** `.env.example` / `app/core/config.py`

`Settings.CORS_ORIGINS` 기본값은 `[]`(안전 기본)인데 `.env.example`은 로컬 프론트 origin을 채워둔다. 의도된 설계("코드에는 origin 리터럴 없음, 예시는 .env.example/문서에만")와 일치 — 신규 개발자가 `.env.example` 복사 시 바로 로컬 프론트 연동 가능. 문제 없음.

**권장 수정 방향:** 유지.
