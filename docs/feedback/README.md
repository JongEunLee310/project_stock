# Feedback

## Directory Purpose

This directory stores records of feedback loops from PRs, CI, local reviews, and retrospectives.

## Included Documents

- `loop-template.md`
- `retrospective-template.md`
- `dogfooding-plan.md`
- `dogfooding-handoffs/`

## Writing Rules

Focus on signal, fix action, re-verification, and documentation update need.

## File Naming Rules

Use descriptive lowercase kebab-case names. Date prefixes may be used for retrospectives.

## Update Criteria

Add a record when CI or review feedback changes future workflow, tests, docs, or decisions.

## Related Directories

- `docs/failures/`
- `docs/knowledge/`
- `docs/decisions/`
