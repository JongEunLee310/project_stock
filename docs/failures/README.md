# Failures

## Directory Purpose

This directory stores failure records for approaches that should not be repeated without understanding why they failed.

## Included Documents

- `FAILURE-000-template.md`

## Writing Rules

Write concrete causes, impact, replacement decisions, and retry conditions. Avoid blame.

## File Naming Rules

Use `FAILURE-NNN-short-title.md`.

## Update Criteria

Add a failure record when a failed approach is likely to be retried or teaches a reusable lesson.
