# Docs

## Directory Purpose

This directory contains reusable documentation for Harness Engineering, workflow knowledge, decisions, failures, feedback records, and archived documents.

## Included Documents

- `harness/`: AI workflow policies.
- `knowledge/`: workflow and domain knowledge.
- `decisions/`: ADRs.
- `failures/`: failure records.
- `feedback/`: feedback loop records.
- `archive/`: deprecated or historical documents.

## Writing Rules

Write durable knowledge, not temporary chat summaries. Separate facts from assumptions. Link related documents when a decision, failure, or workflow change affects future work.

## File Naming Rules

Use descriptive lowercase names for policies and knowledge documents. Use numbered prefixes for ADRs and failure records.

## Update Criteria

Update docs when workflow behavior, project knowledge, review criteria, or governance changes.
